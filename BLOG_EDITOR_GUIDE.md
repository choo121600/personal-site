# 블로그 에디터 사용 가이드

로컬 개발 환경에서 마크다운 기반 블로그 포스트를 작성하고 관리할 수 있는 에디터 시스템입니다.

## 설치

먼저 필요한 라이브러리를 설치하세요:

```bash
npm install gray-matter react-markdown @uiw/react-codemirror @codemirror/lang-markdown @codemirror/theme-one-dark
```

## 디렉토리 구조

시스템이 자동으로 다음 디렉토리를 생성합니다:

```
personal-site/
├── content/
│   └── posts/          # 마크다운 포스트 파일 저장
├── public/
│   └── images/
│       └── YYYY/
│           └── MM/     # 이미지 파일 저장 (연도/월별)
└── app/
    └── admin/
        └── editor/     # 에디터 페이지
```

## 사용 방법

### 1. 에디터 접속

개발 서버를 실행한 후 다음 URL로 접속:

```
http://localhost:3000/admin/editor
```

### 2. 새 포스트 작성

1. "새 포스트" 버튼 클릭
2. 슬러그(URL 경로) 입력 (예: `my-first-post`)
3. 제목, 설명, 날짜 입력
4. 마크다운으로 내용 작성
5. "저장" 버튼 클릭

### 3. 이미지 삽입

**방법 1: 클립보드에서 붙여넣기**
- 이미지를 복사한 후 에디터 영역에 붙여넣기 (Ctrl+V / Cmd+V)
- 이미지가 자동으로 `public/images/YYYY/MM/` 경로에 저장되고 마크다운이 삽입됩니다

**방법 2: 파일 선택**
- "이미지 파일 선택" 버튼 클릭하여 이미지 파일 선택

### 4. 포스트 미리보기

- "미리보기" 버튼을 클릭하여 작성 중인 포스트를 실시간으로 확인
- 다시 "에디터" 버튼을 클릭하여 편집 모드로 돌아가기

### 5. 포스트 관리

- 왼쪽 사이드바에서 기존 포스트 목록 확인
- 포스트 클릭하여 편집
- 포스트에 마우스를 올리면 삭제 버튼 표시

### 6. 포스트 확인

저장된 포스트는 다음 URL로 접근 가능:

```
http://localhost:3000/blog/[슬러그]
```

예: `http://localhost:3000/blog/my-first-post`

## 파일 저장 위치

- **마크다운 파일**: `content/posts/[슬러그].md`
- **이미지 파일**: `public/images/YYYY/MM/[파일명]`

## Frontmatter 형식

각 마크다운 파일은 자동으로 다음 frontmatter를 포함합니다:

```markdown
---
title: 포스트 제목
description: 포스트 설명
date: 2024-01-01
---

포스트 내용...
```

## 특징

- ✅ 파일 저장 시 즉시 반영 (Next.js Hot Reload)
- ✅ 이미지 복사-붙여넣기 자동 처리
- ✅ 실시간 미리보기
- ✅ 마크다운 문법 하이라이팅
- ✅ 다크 모드 지원
- ✅ 로컬 개발 환경에서만 작동 (프로덕션 빌드 시 제외 가능)

## 주의사항

1. **개발 환경 전용**: 이 에디터는 로컬 개발 환경(`npm run dev`)에서만 사용하세요.
2. **파일 저장**: 포스트는 `content/posts/` 디렉토리에 저장되며, Git에 커밋해야 프로덕션에 반영됩니다.
3. **이미지 경로**: 이미지는 `public/images/`에 저장되므로 Git에 커밋해야 합니다.

## 문제 해결

### 이미지가 표시되지 않는 경우
- `public/images/` 디렉토리 권한 확인
- 이미지 파일이 올바른 경로에 저장되었는지 확인

### 포스트가 저장되지 않는 경우
- `content/posts/` 디렉토리 권한 확인
- 슬러그가 올바르게 입력되었는지 확인

### 에디터가 로드되지 않는 경우
- 필요한 라이브러리가 모두 설치되었는지 확인
- 개발 서버를 재시작해보세요
